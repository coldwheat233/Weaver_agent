# 产品需求文档 (PRD) 技能

## PRD 结构
1. **产品概述**: 一句话描述 + 目标用户
2. **用户故事**: As a [role], I want [feature], so that [benefit]
3. **功能需求**: 按优先级 P0/P1/P2 排列
4. **非功能需求**: 性能、安全、可用性
5. **验收标准**: Given/When/Then 格式
6. **里程碑**: 分阶段交付计划

## 需求优先级
- P0 (Must): 没有则产品不可用
- P1 (Should): 重要但有 workaround
- P2 (Nice): 锦上添花
