# 软件架构设计技能

## 常用架构模式
- **分层架构 (Layered)**: 表示层/业务层/持久化层/数据库层
- **微服务 (Microservices)**: 按业务域拆分，独立部署，通过 API/消息通信
- **事件驱动 (Event-Driven)**: 生产者/消费者/消息队列，异步解耦
- **CQRS**: 读写分离，命令模型与查询模型独立
- **六边形架构 (Hexagonal/Ports & Adapters)**: 核心领域与外部依赖通过端口隔离

## 设计关注点
- 可扩展性：水平扩展 > 垂直扩展
- 高可用：冗余、故障转移、健康检查
- 一致性：强一致 vs 最终一致，CAP 定理
- 安全性：鉴权、授权、加密、审计
- 可观测性：日志、指标、追踪

## 输出规范
- 使用 Mermaid graph TD 绘制架构图
- 每个组件明确：职责、技术选型、接口、非功能需求
- 决策需标注备选方案 + 选择理由 + 代价
